# v0.4 Vue Frontend

## 目标

新增 Vue 管理端，复用同一套后端 API，用第二种前端技术栈实现登录和用户管理基础页面。

## 对齐要求

`v0.4` 的目标不是重新设计一套全新的前端业务流，而是用 Vue 实现 `v0.3 React Frontend` 的同一条用户操作链路。页面行为保持一致，但项目结构应体现 Vue 常见开发习惯，避免变成 React 目录的机械翻译。

| 对齐项 | 要求 |
|---|---|
| 页面布局 | 登录页、登录后主布局、侧边菜单、顶部用户信息、首页概览、用户列表和编辑弹窗的整体信息结构应与 React 端保持一致 |
| 用户操作 | 登录、退出、首页/用户管理切换、查询、重置、新增、编辑、逻辑删除、分页切换等操作入口和行为应与 React 端保持一致 |
| 处理逻辑 | 根组件负责启动接线；登录会话、用户管理等状态逻辑优先通过 Vue composables 承载；页面级 SFC 负责模板和交互入口 |
| 项目架构 | `frontend-vue/src` 应使用 Vue 常见分层：`views` 放页面级组件，`layouts` 放管理端外壳，`composables` 放组合式业务逻辑，`api` 放后端请求，`storage` 放本地持久化 |
| 可差异范围 | 组件库、样式、配色、视觉细节和底层本地存储实现可以不同，但不能改变核心页面语义和操作路径 |

## 新增能力

Vue 前端。

## 技术范围

| 类别 | 技术 |
|---|---|
| 运行环境 | Node.js `22.x` |
| 前端 | Vue |
| 开发语言 | JavaScript |
| UI 组件库 | Element UI / Element Plus |
| 通信 | HTTP |
| 后端 | 复用 `v0.2` API |

## 页面范围

| 页面 | 说明 |
|---|---|
| 登录页 | 与 React 登录页保持相同职责：调用登录接口并把登录结果交给根组件保存 |
| 首页 | 与 React 首页保持相同职责：展示当前用户和 JWT 会话恢复结果 |
| 用户列表 | 与 React 用户管理页保持相同职责：分页查询用户、恢复最近查询条件 |
| 用户编辑 | 与 React 用户管理页保持相同职责：通过弹窗新增或编辑用户 |

## 开发步骤

1. 在新终端确认 `node -v` 显示 `v22.x.x`。
2. 创建 `frontend-vue` 项目，使用 JavaScript 模板，不启用 TypeScript。
3. 确认 Vue 版本；如果使用 Vue 3，则引入 Element Plus。
4. 配置 Element UI / Element Plus 全局样式、主题入口和基础布局。
5. 配置开发代理到后端 API。
6. 实现 API 客户端。
7. 实现登录状态管理，并保持根组件统一负责会话恢复、登录成功保存和退出登录。
8. 使用 Element UI / Element Plus 表单、输入框、按钮、卡片等组件实现登录页；登录页不直接操作本地存储。
9. 使用轻量 view state 实现首页概览和用户管理切换，暂不引入 Vue Router，保持与 React 端当前实现一致。
10. 使用 Element UI / Element Plus 表格、分页、弹窗或抽屉组件实现用户列表和编辑表单。
11. 按 Vue 项目习惯拆分目录，优先使用 `layouts`、`views`、`composables` 表达页面、布局和组合式逻辑。
12. 对比 React + Ant Design 与 Vue + Element UI / Element Plus 的目录组织、状态管理和组件使用差异，记录到 README 或单独文档。
13. 更新 `docs/PROGRESS.md`。

## 验收标准

1. Vue 应用可以在 Node.js `22.x` 环境下启动。
2. Vue 业务代码使用 JavaScript 开发，不引入 TypeScript 模板、`tsconfig` 或 `.ts` / `.tsx` 业务代码。
3. Element UI / Element Plus 样式和组件可以正常加载。
4. 浏览器可以登录。
5. 登录后可以查看当前用户。
6. 可以查看用户列表。
7. 可以新增或编辑用户。
8. React 前端仍然可用。
9. 后端 API 未因适配 Vue 产生破坏性变更。
10. Vue 页面布局、操作入口和处理逻辑与 `v0.3` React 管理端保持一致。
11. Vue 前端目录结构体现常见 Vue 开发习惯，而不是简单镜像 React 目录。

## 完成定义

1. Vue 前端可在 Node.js `22.x` 环境下运行。
2. 与后端接口联调通过。
3. README 记录 Vue 前端启动方式、端口、Node.js `22.x`、JavaScript 和 Element UI / Element Plus 组件库要求。
4. 新增和修改代码已补充详细中文注释。
5. 如果需要保存版本，由用户手动提交并手动打 tag：`v0.4-vue-frontend`；Codex 不自动执行提交、tag 或 push。
