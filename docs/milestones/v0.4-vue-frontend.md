# v0.4 Vue Frontend

## 目标

新增 Vue 管理端，复用同一套后端 API，用第二种前端技术栈实现登录和用户管理基础页面。

## 新增能力

Vue 前端。

## 技术范围

| 类别 | 技术 |
|---|---|
| 运行环境 | Node.js `22.x` |
| 前端 | Vue |
| UI 组件库 | Element UI / Element Plus |
| 通信 | HTTP |
| 后端 | 复用 `v0.2` API |

## 页面范围

| 页面 | 说明 |
|---|---|
| 登录页 | 登录并保存 token |
| 首页 | 展示当前用户 |
| 用户列表 | 分页查询用户 |
| 用户编辑 | 新增或编辑用户 |

## 开发步骤

1. 在新终端确认 `node -v` 显示 `v22.x.x`。
2. 创建 `frontend-vue` 项目。
3. 确认 Vue 版本；如果使用 Vue 3，则引入 Element Plus。
4. 配置 Element UI / Element Plus 全局样式、主题入口和基础布局。
5. 配置开发代理到后端 API。
6. 实现 API 客户端。
7. 实现登录状态管理。
8. 使用 Element UI / Element Plus 表单、输入框、按钮、卡片等组件实现登录页和路由守卫。
9. 使用 Element UI / Element Plus 表格、分页、弹窗或抽屉组件实现用户列表和编辑表单。
10. 对比 React + Ant Design 与 Vue + Element UI / Element Plus 的目录组织、状态管理和组件使用差异，记录到 README 或单独文档。
11. 更新 `docs/PROGRESS.md`。

## 验收标准

1. Vue 应用可以在 Node.js `22.x` 环境下启动。
2. Element UI / Element Plus 样式和组件可以正常加载。
3. 浏览器可以登录。
4. 登录后可以查看当前用户。
5. 可以查看用户列表。
6. 可以新增或编辑用户。
7. React 前端仍然可用。
8. 后端 API 未因适配 Vue 产生破坏性变更。

## 完成定义

1. Vue 前端可在 Node.js `22.x` 环境下运行。
2. 与后端接口联调通过。
3. README 记录 Vue 前端启动方式、端口、Node.js `22.x` 和 Element UI / Element Plus 组件库要求。
4. 新增和修改代码已补充详细中文注释。
5. 如果需要保存版本，由用户手动提交并手动打 tag：`v0.4-vue-frontend`；Codex 不自动执行提交、tag 或 push。
