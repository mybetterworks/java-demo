# v0.2 User CRUD

## 目标

在 `v0.1` 登录系统基础上，实现用户管理 CRUD，让系统从“能登录”扩展到“能管理用户”。

## 新增能力

用户管理。

## 技术范围

| 类别 | 技术 |
|---|---|
| 后端 | Spring Boot |
| ORM | MyBatis Plus |
| 数据库 | MySQL |
| 认证 | JWT |

## 功能范围

| 功能 | 说明 |
|---|---|
| 用户分页查询 | 支持按用户名、状态查询 |
| 用户详情 | 根据 ID 查看用户 |
| 创建用户 | 管理端创建用户 |
| 更新用户 | 修改昵称、状态、角色等基础字段 |
| 删除用户 | 优先逻辑删除 |
| 修改密码 | 独立接口，密码继续哈希存储 |

## API 范围

| 方法 | 路径 | 说明 |
|---|---|---|
| `GET` | `/api/users` | 用户分页 |
| `GET` | `/api/users/{id}` | 用户详情 |
| `POST` | `/api/users` | 创建用户 |
| `PUT` | `/api/users/{id}` | 更新用户 |
| `DELETE` | `/api/users/{id}` | 删除用户 |
| `PUT` | `/api/users/{id}/password` | 修改密码 |

## 数据模型扩展

建议新增或明确字段：

| 字段 | 说明 |
|---|---|
| `role` | 简化角色字段，早期可用字符串 |
| `deleted` | 逻辑删除标记 |
| `last_login_at` | 最近登录时间 |

复杂 RBAC 不在本阶段实现，后续可独立扩展。

## 开发步骤

1. 读取 `v0.1` 完成状态，确认登录和数据库可用。
2. 扩展用户表字段。
3. 实现用户分页查询。
4. 实现用户详情。
5. 实现创建、更新、删除接口。
6. 实现修改密码接口。
7. 给用户管理接口加 JWT 鉴权。
8. 添加参数校验和常见异常处理。
9. 添加接口测试或服务层测试。
10. 为新增和修改的 Java、配置、SQL、测试代码补充详细中文注释。
11. 更新 `README.md` 和 `docs/PROGRESS.md`。

## 验收标准

1. 未登录不能访问用户管理接口。
2. 登录后可以分页查询用户。
3. 可以创建新用户。
4. 可以修改用户基础信息。
5. 可以逻辑删除用户。
6. 已删除用户不会出现在默认列表中。
7. 修改密码后旧密码不能登录，新密码可以登录。
