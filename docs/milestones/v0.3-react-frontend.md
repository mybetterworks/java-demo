# v0.3 React Frontend

## 目标

新增 React 管理端，让用户可以通过浏览器完成登录和用户管理基础操作。

## 新增能力

React 前端。

## 技术范围

| 类别 | 技术 |
|---|---|
| 运行环境 | Node.js `22.x` |
| 前端 | React |
| 开发语言 | TypeScript |
| UI 组件库 | Ant Design |
| 本地缓存 | IndexedDB |
| 通信 | HTTP |
| 后端 | 复用 `v0.2` API |

## 页面范围

| 页面 | 说明 |
|---|---|
| 登录页 | 输入用户名密码，登录后保存 token |
| 首页 | 展示当前登录用户 |
| 用户列表 | 分页查询用户 |
| 用户编辑 | 新增或编辑用户 |

## IndexedDB 使用范围

本阶段不要把 IndexedDB 用复杂。

建议只保存：

| 数据 | 说明 |
|---|---|
| `auth_session` | token、用户名、登录时间 |
| `recent_users_query` | 最近一次用户列表查询条件 |

安全提示：真实生产环境中 token 存储策略需要更谨慎，本项目早期以学习和验证为主。

## 开发步骤

1. 在新终端确认 `node -v` 显示 `v22.x.x`。
2. 创建 `frontend-react` 项目，使用 Vite + React + TypeScript 模板。
3. 引入 Ant Design，配置全局样式、主题入口和基础布局。
4. 配置本地开发代理到后端 API。
5. 实现 API 客户端和统一错误处理。
6. 实现 IndexedDB 轻量封装。
7. 使用 Ant Design Form、Input、Button、Card 等组件实现登录页。
8. 实现路由守卫或登录态检查。
9. 使用 Ant Design Layout、Table、Pagination 等组件实现用户列表页。
10. 使用 Ant Design Form、Modal 或 Drawer 实现新增和编辑用户表单。
11. 补充前端启动说明，明确 Node.js `22.x`、TypeScript 和 Ant Design 为默认前端要求。
12. 更新 `README.md` 和 `docs/PROGRESS.md`。
