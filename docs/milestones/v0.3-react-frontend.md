# v0.3 React Frontend

## 目标

新增 React 管理端，让用户可以通过浏览器完成登录和用户管理基础操作。

## 新增能力

React 前端。

## 技术范围

| 类别 | 技术 |
|---|---|
| 运行环境 | Node.js `22.x` |
| 前端 | React |
| UI 组件库 | Ant Design |
| 本地缓存 | IndexedDB |
| 通信 | HTTP |
| 后端 | 复用 `v0.2` API |

## 页面范围

| 页面 | 说明 |
|---|---|
| 登录页 | 输入用户名密码，登录后保存 token |
| 首页 | 展示当前登录用户 |
| 用户列表 | 分页查询用户 |
| 用户编辑 | 新增或编辑用户 |

## IndexedDB 使用范围

本阶段不要把 IndexedDB 用复杂。

建议只保存：

| 数据 | 说明 |
|---|---|
| `auth_session` | token、用户名、登录时间 |
| `recent_users_query` | 最近一次用户列表查询条件 |

安全提示：真实生产环境中 token 存储策略需要更谨慎，本项目早期以学习和验证为主。

## 开发步骤

1. 在新终端确认 `node -v` 显示 `v22.x.x`。
2. 创建 `frontend-react` 项目。
3. 引入 Ant Design，配置全局样式、主题入口和基础布局。
4. 配置本地开发代理到后端 API。
5. 实现 API 客户端和统一错误处理。
6. 实现 IndexedDB 轻量封装。
7. 使用 Ant Design Form、Input、Button、Card 等组件实现登录页。
8. 实现路由守卫或登录态检查。
9. 使用 Ant Design Layout、Table、Pagination 等组件实现用户列表页。
10. 使用 Ant Design Form、Modal 或 Drawer 实现新增和编辑用户表单。
11. 补充前端启动说明，明确 Node.js `22.x` 和 Ant Design 为默认前端要求。
12. 更新 `README.md` 和 `docs/PROGRESS.md`。

## 验收标准

1. React 应用可以在 Node.js `22.x` 环境下启动。
2. Ant Design 样式和组件可以正常加载。
3. 浏览器可以登录。
4. 登录后可以查看当前用户。
5. 刷新页面后登录状态可以从 IndexedDB 恢复。
6. 可以查看用户列表。
7. 可以新增或编辑用户。
8. 后端登录和用户管理接口未回归。

## 完成定义

1. React 前端可在 Node.js `22.x` 环境下运行。
2. 与后端接口联调通过。
3. README 记录前端启动方式、端口、Node.js `22.x` 和 Ant Design 组件库要求。
4. 新增和修改代码已补充详细中文注释。
5. 如果需要保存版本，由用户手动提交并手动打 tag：`v0.3-react-frontend`；Codex 不自动执行提交、tag 或 push。
