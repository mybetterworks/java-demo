# v0.8 WebSocket

## 目标

引入 WebSocket，实现服务端向前端推送基础消息，例如任务分配、通知未读数变化、用户状态变化或系统广播。

## 新增能力

实时通信。

## 技术范围

| 类别 | 技术 |
|---|---|
| 后端 | WebSocket |
| 前端 | React 和 Vue WebSocket 客户端 |
| 网关 | WebSocket 转发 |

## 功能范围

| 功能 | 说明 |
|---|---|
| WebSocket 连接 | 登录后建立连接 |
| 用户身份识别 | 使用 token 识别用户 |
| 任务通知 | `task-service` 创建或更新任务后触发通知推送 |
| 未读数量变化 | `notification-service` 推送当前用户未读数变化 |
| 系统通知 | 后端主动推送消息 |
| 在线状态 | 简单记录连接状态 |

## 日志要求

继承 `v0.5.2` 后端运行日志基线。本阶段需要重点记录 WebSocket 握手、鉴权失败、连接建立、连接断开、断线重连、指定用户推送、广播推送和推送失败，便于在控制台和日志文件中观察实时通信链路。

## 开发步骤

1. 明确 WebSocket endpoint 所属服务，优先放在 `notification-service`，由它统一推送站内信和未读数变化。
2. 定义消息格式。
3. 使用 JWT 或握手参数识别用户。
4. Gateway 支持 WebSocket 转发。
5. React 前端接入 WebSocket。
6. Vue 前端接入 WebSocket。
7. 任务分配、任务状态变化、通知创建或手动接口触发推送。
8. 增加断线重连和错误提示的基础逻辑。
9. 在连接、鉴权、推送、断开和异常路径补充必要日志。
10. 更新 README 和 `docs/PROGRESS.md`。

## 验收标准

1. 登录后前端可以建立 WebSocket 连接。
2. 未登录或无效 token 不能建立有效连接。
3. `notification-service` 可以向指定用户推送任务或通知消息。
4. 未读通知数量变化可以推送到 React 和 Vue。
5. 后端可以广播系统消息。
6. React 和 Vue 都能收到推送。
7. Gateway 转发 WebSocket 正常。
8. 控制台和文件日志可以观察连接建立、鉴权失败、推送成功和推送失败。

## 完成定义

1. WebSocket 推送链路可验证。
2. 两套前端均完成接入。
3. 消息格式已记录。
4. WebSocket 关键日志可验证，且不打印完整 JWT 或 Authorization header。
5. 新增和修改代码已补充详细中文注释。
6. 如果需要保存版本，由用户手动提交并手动打 tag：`v0.8-websocket`；Codex 不自动执行提交、tag 或 push。
