# v0.8 WebSocket

## 目标

引入 WebSocket，实现服务端向前端推送基础消息，例如登录通知、用户状态变化或系统广播。

## 新增能力

实时通信。

## 技术范围

| 类别 | 技术 |
|---|---|
| 后端 | WebSocket |
| 前端 | React 和 Vue WebSocket 客户端 |
| 网关 | WebSocket 转发 |

## 功能范围

| 功能 | 说明 |
|---|---|
| WebSocket 连接 | 登录后建立连接 |
| 用户身份识别 | 使用 token 识别用户 |
| 系统通知 | 后端主动推送消息 |
| 在线状态 | 简单记录连接状态 |

## 开发步骤

1. 后端新增 WebSocket endpoint。
2. 定义消息格式。
3. 使用 JWT 或握手参数识别用户。
4. Gateway 支持 WebSocket 转发。
5. React 前端接入 WebSocket。
6. Vue 前端接入 WebSocket。
7. 登录、用户变更或手动接口触发通知。
8. 增加断线重连和错误提示的基础逻辑。
9. 更新 README 和 `docs/PROGRESS.md`。

## 验收标准

1. 登录后前端可以建立 WebSocket 连接。
2. 未登录或无效 token 不能建立有效连接。
3. 后端可以向指定用户推送消息。
4. 后端可以广播系统消息。
5. React 和 Vue 都能收到推送。
6. Gateway 转发 WebSocket 正常。

## 完成定义

1. WebSocket 推送链路可验证。
2. 两套前端均完成接入。
3. 消息格式已记录。
4. 新增和修改代码已补充详细中文注释。
5. 如果需要保存版本，由用户手动提交并手动打 tag：`v0.8-websocket`；Codex 不自动执行提交、tag 或 push。
