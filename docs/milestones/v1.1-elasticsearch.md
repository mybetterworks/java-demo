# v1.1 Elasticsearch

## 目标

引入 Elasticsearch，为用户和审计记录提供全文检索能力。

## 新增能力

全文检索。

## 技术范围

| 类别 | 技术 |
|---|---|
| 搜索 | Elasticsearch |
| 数据来源 | 用户表、登录审计表 |
| 同步方式 | 同步写入或消息异步同步 |

## 功能范围

| 功能 | 说明 |
|---|---|
| 用户索引 | 支持 username、nickname 搜索 |
| 审计索引 | 支持登录时间、用户名、结果搜索 |
| 数据同步 | 用户变更后同步 ES |
| 搜索 API | 后端提供统一搜索接口 |

## 开发步骤

1. 新增 Elasticsearch Docker Compose。
2. 定义用户索引 mapping。
3. 定义审计索引 mapping。
4. 实现用户数据同步到 ES。
5. 实现审计数据同步到 ES。
6. 实现搜索接口。
7. React 前端增加搜索入口。
8. Vue 前端增加搜索入口。
9. 记录重建索引命令。
10. 更新 README 和 `docs/PROGRESS.md`。

## 验收标准

1. Elasticsearch 可以启动。
2. 用户数据可以写入索引。
3. 审计数据可以写入索引。
4. 可以按关键字搜索用户。
5. 可以搜索登录审计记录。
6. 用户更新后搜索结果会同步变化。

## 完成定义

1. 搜索能力可验证。
2. 索引 mapping 和重建方式已记录。
3. 新增和修改代码已补充详细中文注释。
4. 如果需要保存版本，由用户手动提交并手动打 tag：`v1.1-elasticsearch`；Codex 不自动执行提交、tag 或 push。
