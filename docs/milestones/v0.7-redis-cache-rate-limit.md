# v0.7 Redis Cache And Rate Limit

## 目标

引入 Redis，用于缓存、登录辅助能力和基础限流，为后续 Redis Cluster 做准备。

## 新增能力

缓存与限流。

## 技术范围

| 类别 | 技术 |
|---|---|
| 缓存 | Redis |
| 集群目标 | Redis Cluster |
| 网关 | Gateway 限流或业务限流 |

## 功能范围

| 功能 | 说明 |
|---|---|
| 当前用户缓存 | 缓存用户基础信息 |
| 登录失败计数 | 防止暴力尝试 |
| 简单限流 | 针对登录或用户查询接口 |
| Redis Cluster 预案 | 准备集群 compose 或说明 |

## 开发步骤

1. 新增 Redis 单节点 Docker Compose。
2. 后端接入 Redis 客户端。
3. 实现当前用户信息缓存。
4. 实现登录失败计数和短时间锁定。
5. 在 Gateway 或后端实现基础限流。
6. 增加缓存失效逻辑，例如用户更新后删除缓存。
7. 准备 Redis Cluster compose 或单独文档。
8. 更新 README 和 `docs/PROGRESS.md`。

## 验收标准

1. Redis 可以启动。
2. 用户信息查询可以命中缓存。
3. 用户信息更新后缓存会失效。
4. 多次错误登录会触发限制。
5. 限流规则可通过接口验证。
6. 不启动 Redis 时系统有明确错误提示或降级说明。

## 完成定义

1. 缓存能力可验证。
2. 限流能力可验证。
3. Redis 配置和端口已记录。
4. 新增和修改代码已补充详细中文注释。
5. 如果使用 Git，提交并打 tag：`v0.7-redis-cache-rate-limit`。
