# v1.4 MySQL Replication

## 目标

引入 MySQL 主从集群和读写分离，验证业务数据库高可用与读写拆分的基本模式。

## 新增能力

MySQL 主从和读写分离。

## 技术范围

| 类别 | 技术 |
|---|---|
| 数据库 | MySQL 主从复制 |
| 数据源 | 动态数据源或读写分离组件 |
| 部署 | Docker Compose |

## 功能范围

| 功能 | 说明 |
|---|---|
| 主库写入 | 注册、用户修改写主库 |
| 从库读取 | 查询接口优先读从库 |
| 复制验证 | 主库写入后从库可见 |
| 故障演练 | 记录主从异常时的行为 |

## 容器策略

遵守 `docs/decisions/0010-docker-service-containerization.md`。

| 节点 | 容器要求 |
|---|---|
| MySQL 主库 | 使用独立容器，例如 `java-demo-mysql-primary` |
| MySQL 从库 | 每个从库独立容器，例如 `java-demo-mysql-replica-1` |
| 初始化脚本 | 主从初始化 SQL 和配置文件放在项目目录，不写入 Docker Desktop 全局配置 |

## 开发步骤

1. 新增 MySQL 主从 Docker Compose，主库和每个从库都使用独立容器。
2. 编写主从初始化脚本。
3. 配置后端多数据源。
4. 将写操作路由到主库。
5. 将读操作路由到从库。
6. 增加复制状态检查说明。
7. 验证主从延迟对业务查询的影响。
8. 更新 README 和 `docs/PROGRESS.md`。

## 验收标准

1. MySQL 主库和从库可以启动。
2. 主从复制状态正常。
3. 注册用户写入主库。
4. 用户列表读取从库。
5. 主库写入后从库能看到数据。
6. 停止从库时系统行为有明确记录。
7. 主库、从库容器名、端口、volume、network 和复制关系已记录。

## 完成定义

1. 主从复制可验证。
2. 读写分离可验证。
3. 故障行为已记录。
4. 新增和修改代码已补充详细中文注释。
5. 如果需要保存版本，由用户手动提交并手动打 tag：`v1.4-mysql-replication`；Codex 不自动执行提交、tag 或 push。
