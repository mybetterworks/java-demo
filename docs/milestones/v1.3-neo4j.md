# v1.3 Neo4j

## 目标

引入 Neo4j，用图数据库表达用户、角色、组织之间的关系，验证关系查询和图谱建模。

## 新增能力

图关系查询。

## 技术范围

| 类别 | 技术 |
|---|---|
| 图数据库 | Neo4j |
| 业务场景 | 用户、角色、组织关系 |

## 图模型建议

节点：

| 节点 | 说明 |
|---|---|
| `User` | 用户 |
| `Role` | 角色 |
| `Org` | 组织 |

关系：

| 关系 | 说明 |
|---|---|
| `HAS_ROLE` | 用户拥有角色 |
| `BELONGS_TO` | 用户属于组织 |
| `ORG_PARENT` | 组织上下级 |

## 容器策略

遵守 `docs/decisions/0010-docker-service-containerization.md`。

| 阶段 | 容器要求 |
|---|---|
| 单节点验证 | 使用 Docker Desktop 拉取 Neo4j 镜像并运行 `java-demo-neo4j-1` 独立容器 |
| 数据持久化 | data、logs、plugins 等 volume 独立命名并记录用途 |
| 后续扩展 | 如未来验证 Neo4j 集群，每个 Neo4j 节点独立容器 |

## 开发步骤

1. 新增 Neo4j Docker Compose，记录镜像版本、容器名、端口、volume、network 和健康检查。
2. 后端接入 Neo4j 客户端或 Spring Data Neo4j。
3. 定义图模型和同步策略。
4. 用户创建或更新时同步图节点。
5. 实现用户关系查询接口。
6. 实现组织关系查询接口。
7. 前端展示简单关系结果。
8. 记录 Cypher 示例。
9. 更新 README 和 `docs/PROGRESS.md`。

## 验收标准

1. Neo4j 可以启动并访问控制台。
2. 用户、角色、组织节点可以写入。
3. 用户和角色关系可以查询。
4. 用户和组织关系可以查询。
5. 前端可以展示基础关系信息。
6. Neo4j 单节点和后续集群节点容器边界已记录。
