# v1.9 Jenkins CI CD

## 目标

引入 Jenkins，建立从代码提交到构建、测试、镜像打包和部署的基础 CI/CD 流水线。

## 新增能力

CI/CD。

## 技术范围

| 类别 | 技术 |
|---|---|
| CI/CD | Jenkins |
| 构建 | Maven Wrapper、前端构建 |
| 镜像 | Docker |
| 部署 | Docker Compose 或 Kubernetes |

## 流水线范围

| 阶段 | 说明 |
|---|---|
| Checkout | 拉取代码 |
| Backend Test | 后端测试 |
| Frontend Build | React/Vue 构建 |
| Docker Build | 构建镜像 |
| Deploy | 部署到 Compose 或 K8s |

## 开发步骤

1. 新增 Jenkins Docker Compose。
2. 编写 `Jenkinsfile`。
3. 配置后端测试阶段。
4. 配置前端构建阶段。
5. 配置 Docker 镜像构建阶段。
6. 配置镜像 tag 策略。
7. 配置部署阶段，优先支持本地演示环境。
8. 记录 Jenkins 初始化、插件和凭据配置。
9. 运行一次完整流水线。
10. 更新 README 和 `docs/PROGRESS.md`。

## 验收标准

1. Jenkins 可以启动并访问控制台。
2. 流水线可以拉取当前仓库代码。
3. 后端测试阶段可执行。
4. 前端构建阶段可执行。
5. Docker 镜像可以构建。
6. 部署阶段可以完成一次演示部署。
7. 失败阶段能在 Jenkins 中看到明确日志。

## 完成定义

1. 一条完整 CI/CD 流水线可验证。
2. Jenkinsfile 已提交。
3. 初始化和凭据配置已文档化。
4. 新增和修改代码已补充详细中文注释。
5. 如果使用 Git，提交并打 tag：`v1.9-jenkins-ci-cd`。
