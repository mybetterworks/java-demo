# v1.9 Jenkins CI CD

## 目标

引入 Jenkins，建立从代码提交到构建、测试、镜像打包和部署的基础 CI/CD 流水线。

## 新增能力

CI/CD。

## 技术范围

| 类别 | 技术 |
|---|---|
| CI/CD | Jenkins |
| 构建 | Maven Wrapper、前端构建 |
| 镜像 | Docker |
| 部署 | Docker Compose 或 Kubernetes |

## 流水线范围

| 阶段 | 说明 |
|---|---|
| Checkout | 拉取代码 |
| Backend Test | 用户服务、任务服务、通知服务和 Gateway 测试 |
| Frontend Build | React/Vue 构建 |
| Docker Build | 构建用户服务、任务服务、通知服务、Gateway 和前端镜像 |
| Deploy | 部署到 Compose 或 K8s |

## 容器策略

遵守 `docs/decisions/0010-docker-service-containerization.md`。

| 服务 | 容器要求 |
|---|---|
| Jenkins controller | 使用 Docker Desktop 拉取 Jenkins 镜像并运行 `java-demo-jenkins-1` 独立容器 |
| Jenkins agent | 如后续使用 agent，agent 与 controller 分开容器 |
| 构建依赖 | Docker、Maven、Node.js 等构建环境通过 Jenkinsfile 或 agent 镜像明确描述，不与基础设施服务混装 |

## 开发步骤

1. 新增 Jenkins Docker Compose，Jenkins controller 使用独立容器。
2. 编写 `Jenkinsfile`。
3. 配置后端测试阶段，覆盖 `java-demo-app`、`task-service`、`notification-service` 和 `gateway`。
4. 配置前端构建阶段。
5. 配置 Docker 镜像构建阶段，分别构建多服务镜像。
6. 配置镜像 tag 策略。
7. 配置部署阶段，优先支持本地演示环境。
8. 记录 Jenkins 初始化、插件和凭据配置。
9. 运行一次完整流水线。
10. 更新 README 和 `docs/PROGRESS.md`。

## 验收标准

1. Jenkins 可以启动并访问控制台。
2. 流水线可以拉取当前仓库代码。
3. 后端测试阶段可执行，并覆盖用户、任务、通知和 Gateway。
4. 前端构建阶段可执行。
5. 多服务 Docker 镜像可以构建。
6. 部署阶段可以完成一次演示部署。
7. 失败阶段能在 Jenkins 中看到明确日志。
8. Jenkins controller、agent 和构建依赖容器边界已记录。

## 完成定义

1. 一条完整 CI/CD 流水线可验证。
2. Jenkinsfile 已创建并文档化；如需进入 Git 历史，由用户手动提交。
3. 初始化和凭据配置已文档化。
4. 新增和修改代码已补充详细中文注释。
5. 如果需要保存版本，由用户手动提交并手动打 tag：`v1.9-jenkins-ci-cd`；Codex 不自动执行提交、tag 或 push。
