# v0.1 MVP Login

## 目标

实现第一个最小可运行版本：用户可以注册、登录、获取 JWT，并通过 JWT 查询当前登录用户。

本阶段只追求闭环，不追求微服务拆分。

## 新增能力

最小登录系统。

## 技术范围

| 类别 | 技术 |
|---|---|
| 后端 | Spring Boot |
| ORM | MyBatis Plus |
| 数据库 | MySQL Docker 单节点 |
| 认证 | JWT |
| 接口文档 | Springdoc OpenAPI、Swagger UI |
| 构建 | Maven Wrapper、Maven `3.9.16` |

## 建议模块

```text
backend
└─ app
   ├─ controller
   ├─ service
   ├─ mapper
   ├─ entity
   ├─ dto
   ├─ config
   └─ security
```

后续再按需要演进为 `auth-service`、`user-service`、`gateway`。

## 数据模型

最小用户表建议字段：

| 字段 | 说明 |
|---|---|
| `id` | 主键 |
| `username` | 登录名，唯一 |
| `password_hash` | 密码哈希 |
| `nickname` | 昵称 |
| `status` | 用户状态 |
| `created_at` | 创建时间 |
| `updated_at` | 更新时间 |

## API 范围

| 方法 | 路径 | 说明 |
|---|---|---|
| `POST` | `/api/auth/register` | 注册用户 |
| `POST` | `/api/auth/login` | 登录并返回 JWT |
| `GET` | `/api/users/me` | 获取当前用户 |
| `GET` | `/api/health` | 健康检查 |
| `GET` | `/v3/api-docs` | OpenAPI JSON |
| `GET` | `/swagger-ui.html` | Swagger UI 页面 |

## 开发步骤

1. 确认 Git 仓库状态；如需保存版本，由用户手动提交、手动打 tag 和手动推送。
2. 确认 JDK `17.0.19` 和 Maven `3.9.16` 可用。
3. 确认 Maven 本地仓库使用 `D:\software\maven_download`。
4. 创建 Maven Wrapper，统一后续构建入口。
5. 创建后端 Spring Boot 项目骨架。
6. 创建 MySQL Docker Compose 文件。
7. 创建数据库初始化脚本或迁移脚本。
8. 实现统一响应结构。
9. 实现参数校验和全局异常处理。
10. 实现用户注册，密码必须哈希保存。
11. 实现登录，校验密码并签发 JWT。
12. 实现 JWT 解析和当前用户接口。
13. 添加最小单元测试或接口测试。
14. 为新增 Java 代码、配置、SQL 和 Docker Compose 补充详细中文注释。
15. 更新 `README.md` 和 `docs/PROGRESS.md`。
