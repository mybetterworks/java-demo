# v1.2 InfluxDB

## 目标

引入 InfluxDB，保存登录次数、任务创建量、通知发送量、接口耗时等时序数据，为监控和趋势分析做准备。

## 新增能力

时序统计。

## 技术范围

| 类别 | 技术 |
|---|---|
| 时序数据库 | InfluxDB |
| 数据场景 | 登录趋势、任务趋势、通知趋势、接口耗时、用户操作统计 |

## 功能范围

| 功能 | 说明 |
|---|---|
| 登录指标 | 记录登录成功、失败次数 |
| 任务指标 | 记录任务创建、完成、取消次数 |
| 通知指标 | 记录通知创建、已读、未读数量变化 |
| 接口耗时 | 记录关键接口耗时 |
| 趋势查询 | 查询一段时间内的指标 |
| 前端展示 | 简单趋势图或统计卡片 |

## 容器策略

遵守 `docs/decisions/0010-docker-service-containerization.md`。

| 阶段 | 容器要求 |
|---|---|
| 单节点验证 | 使用 Docker Desktop 拉取 InfluxDB 镜像并运行 `java-demo-influxdb-1` 独立容器 |
| 辅助初始化 | bucket、org、token 初始化脚本或 job 与 InfluxDB 服务容器边界分清 |
| 后续扩展 | 如增加 Telegraf 等采集组件，采集组件独立容器运行 |

## 开发步骤

1. 新增 InfluxDB Docker Compose，记录镜像版本、容器名、端口、volume、network 和健康检查。
2. 后端接入 InfluxDB 客户端。
3. 登录接口写入登录统计点。
4. 任务创建、完成、取消时写入任务统计点。
5. 通知创建、已读时写入通知统计点。
6. 使用拦截器或过滤器记录接口耗时，覆盖用户、任务、通知核心接口。
7. 实现指标查询 API。
8. React 前端展示简单趋势。
9. Vue 前端展示简单趋势。
10. 记录数据保留策略和 bucket 配置。
11. 更新 README 和 `docs/PROGRESS.md`。

## 验收标准

1. InfluxDB 可以启动。
2. 登录成功和失败会写入时序数据。
3. 任务创建、完成、取消会写入时序数据。
4. 通知创建、已读会写入时序数据。
5. 关键接口耗时会写入时序数据。
6. 后端可以查询趋势数据。
7. 前端可以展示基础趋势。
8. InfluxDB 及后续采集组件容器边界已记录。

## 完成定义

1. 时序写入和查询可验证。
2. 指标命名和字段已记录。
3. 新增和修改代码已补充详细中文注释。
4. 如果需要保存版本，由用户手动提交并手动打 tag：`v1.2-influxdb`；Codex 不自动执行提交、tag 或 push。
