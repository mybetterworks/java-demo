# v1.2 InfluxDB

## 目标

引入 InfluxDB，保存登录次数、接口耗时等时序数据，为监控和趋势分析做准备。

## 新增能力

时序统计。

## 技术范围

| 类别 | 技术 |
|---|---|
| 时序数据库 | InfluxDB |
| 数据场景 | 登录趋势、接口耗时、用户操作统计 |

## 功能范围

| 功能 | 说明 |
|---|---|
| 登录指标 | 记录登录成功、失败次数 |
| 接口耗时 | 记录关键接口耗时 |
| 趋势查询 | 查询一段时间内的指标 |
| 前端展示 | 简单趋势图或统计卡片 |

## 开发步骤

1. 新增 InfluxDB Docker Compose。
2. 后端接入 InfluxDB 客户端。
3. 登录接口写入登录统计点。
4. 使用拦截器或过滤器记录接口耗时。
5. 实现指标查询 API。
6. React 前端展示简单趋势。
7. Vue 前端展示简单趋势。
8. 记录数据保留策略和 bucket 配置。
9. 更新 README 和 `docs/PROGRESS.md`。

## 验收标准

1. InfluxDB 可以启动。
2. 登录成功和失败会写入时序数据。
3. 关键接口耗时会写入时序数据。
4. 后端可以查询趋势数据。
5. 前端可以展示基础趋势。

## 完成定义

1. 时序写入和查询可验证。
2. 指标命名和字段已记录。
3. 新增和修改代码已补充详细中文注释。
4. 如果需要保存版本，由用户手动提交并手动打 tag：`v1.2-influxdb`；Codex 不自动执行提交、tag 或 push。
