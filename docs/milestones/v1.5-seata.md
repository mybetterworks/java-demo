# v1.5 Seata

## 目标

引入 Seata 3 节点集群，验证跨服务分布式事务。此阶段需要服务边界足够清晰，至少涉及两个后端服务。

## 新增能力

分布式事务。

## 技术范围

| 类别 | 技术 |
|---|---|
| 分布式事务 | Seata |
| 注册配置 | Nacos |
| 数据库 | MySQL |
| 服务 | Auth/User 或 User/Audit |

## 业务场景建议

创建用户时同时完成：

1. 用户服务写入用户表。
2. 认证服务写入认证账号或初始化登录凭据。
3. 审计服务写入操作记录。

任一环节失败时整体回滚。

## 开发步骤

1. 确认服务拆分状态是否满足 Seata 验证。
2. 新增 Seata 3 节点 Docker Compose。
3. Seata 接入 Nacos 注册和配置。
4. 初始化 Seata 所需数据库表。
5. 后端服务接入 Seata 客户端。
6. 实现跨服务事务业务接口。
7. 添加一个可控失败开关，用于验证回滚。
8. 验证提交和回滚路径。
9. 更新 README 和 `docs/PROGRESS.md`。

## 验收标准

1. Seata 3 个节点可以启动。
2. Seata 服务可以注册到 Nacos。
3. 正常创建用户时多个服务数据一致。
4. 制造异常时多个服务数据一起回滚。
5. 事务日志和业务日志可追踪。

## 完成定义

1. 分布式事务提交可验证。
2. 分布式事务回滚可验证。
3. Seata 集群部署和故障排查已记录。
4. 新增和修改代码已补充详细中文注释。
5. 如果使用 Git，提交并打 tag：`v1.5-seata`。
