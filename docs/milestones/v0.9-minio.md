# v0.9 MinIO

## 目标

引入 MinIO，支持用户头像或附件上传，验证对象存储在微服务系统中的基本用法。

## 新增能力

文件对象存储。

## 技术范围

| 类别 | 技术 |
|---|---|
| 文件系统 | MinIO |
| 后端 | 文件上传接口 |
| 前端 | 头像上传组件 |

## 功能范围

| 功能 | 说明 |
|---|---|
| Bucket 初始化 | 创建用户头像 bucket |
| 文件上传 | 上传头像或附件 |
| 文件访问 | 返回可访问 URL 或代理下载 |
| 用户头像 | 用户资料关联头像地址 |

## 开发步骤

1. 新增 MinIO Docker Compose。
2. 后端接入 MinIO SDK。
3. 创建文件上传接口。
4. 增加文件类型和大小校验。
5. 用户表增加头像字段。
6. React 前端增加头像上传。
7. Vue 前端增加头像上传。
8. 编写 MinIO 控制台访问说明。
9. 更新 README 和 `docs/PROGRESS.md`。

## 验收标准

1. MinIO 可以启动并访问控制台。
2. 后端可以上传文件到 MinIO。
3. 非法文件类型或过大文件会被拒绝。
4. 用户头像上传后可以在用户详情中看到。
5. React 和 Vue 前端均可上传头像。

## 完成定义

1. 文件上传和访问可验证。
2. MinIO 配置已文档化。
3. 用户管理功能未回归。
4. 新增和修改代码已补充详细中文注释。
5. 如果需要保存版本，由用户手动提交并手动打 tag：`v0.9-minio`；Codex 不自动执行提交、tag 或 push。
