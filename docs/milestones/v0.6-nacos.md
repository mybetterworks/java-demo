# v0.6 Nacos

## 目标

引入 Nacos，用于服务注册发现和配置管理，让网关和后端服务从静态地址转向服务名路由。

## 新增能力

服务注册与配置中心。

## 技术范围

| 类别 | 技术 |
|---|---|
| 注册中心 | Nacos |
| 配置中心 | Nacos Config |
| 服务治理 | Spring Cloud |

## 架构变化

```text
Gateway
  -> Nacos service discovery
    -> java-demo-app
    -> task-service
    -> notification-service
```

本阶段优先实现 Nacos 单节点可运行，再保留集群扩展 compose 计划。集群化可以在后续部署阶段继续强化。

`v0.5.1` 已规划任务和通知两个业务服务，因此本阶段不只验证单个后端注册，而是让 Gateway、用户、任务、通知服务都进入注册发现体系。这样后续服务扩容、下线和路由切换可以直接在真实业务链路中观察。

`v0.5.2` 已规划后端运行日志基线，因此本阶段接入 Nacos 时需要继续保留并强化关键日志，重点观察服务注册、配置读取、服务名调用、配置刷新和服务不可用场景。

`v0.5.3` 已规划 React/Vue 任务和通知前端，因此本阶段除接口脚本外，还需要回归两套前端经 Gateway 访问用户、任务和通知功能是否正常。Nacos 改造本身通常不新增页面，但服务发现路由变化不能破坏已有前端操作路径。

## 容器策略

遵守 `docs/decisions/0010-docker-service-containerization.md`。

| 阶段 | 容器要求 |
|---|---|
| 单节点验证 | 使用 Docker Desktop 拉取 Nacos 镜像并运行 `java-demo-nacos-1` 独立容器 |
| 集群扩展 | Nacos 每个节点独立容器，例如 `java-demo-nacos-1`、`java-demo-nacos-2`、`java-demo-nacos-3` |
| 依赖服务 | 如果 Nacos 集群需要数据库，数据库继续使用独立 MySQL 容器，不与 Nacos 合并 |

## 开发步骤

1. 新增 Nacos Docker Compose 文件，单节点容器和后续集群容器规划分开记录。
2. `java-demo-app`、`task-service`、`notification-service` 和 `gateway` 接入服务注册。
3. Gateway 从静态地址路由改为服务名路由，例如用户、任务、通知分别按服务名转发。
4. `task-service` 调用 `java-demo-app`、`task-service` 调用 `notification-service` 的链路从静态地址迁移为服务发现调用。
5. 把部分配置迁移到 Nacos Config，例如 JWT 配置、任务状态开关、通知发送开关或登录限制。
6. 在服务注册、配置加载、配置刷新、服务名调用成功和调用失败处补充必要日志。
7. 编写本地启动顺序说明。
8. 验证服务上下线对网关路由和服务间调用的影响。
9. 回归 React 和 Vue 的登录、任务管理、通知中心，确认前端经 Gateway 调用服务发现路由正常。
10. 准备 Nacos 集群化草案，但不强行复杂化本阶段。
11. 更新 README 和 `docs/PROGRESS.md`。

## 验收标准

1. Nacos 可以启动并访问控制台。
2. Gateway、`java-demo-app`、`task-service`、`notification-service` 可以注册到 Nacos。
3. Gateway 可以通过服务名转发用户、任务和通知请求。
4. `task-service` 可以通过服务发现调用用户服务和通知服务。
5. 修改 Nacos 配置后，服务可以读取到配置。
6. 控制台和文件日志可以观察服务注册、配置读取、服务名调用和失败原因。
7. 登录、用户管理、任务、通知、React、Vue 功能未回归。
8. React 和 Vue 任务/通知页面仍可经 Gateway 正常访问后端。
9. Nacos 容器名称、镜像版本、端口、volume、network 和健康检查已记录。

## 完成定义

1. Nacos 服务注册可用。
2. Nacos 配置读取可用。
3. README 记录 Nacos 启动和访问方式。
4. Nacos 相关关键流程日志可在控制台和文件中观察，且不打印敏感信息。
5. 新增和修改代码已补充详细中文注释。
6. 服务间调用仍可保留当前 HTTP/REST 方式，本版本不同时引入 OpenFeign 或 Dubbo，避免范围过大。
7. 如果需要保存版本，由用户手动提交并手动打 tag：`v0.6-nacos`；Codex 不自动执行提交、tag 或 push。
