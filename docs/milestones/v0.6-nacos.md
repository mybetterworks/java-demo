# v0.6 Nacos

## 目标

引入 Nacos，用于服务注册发现和配置管理，让网关和后端服务从静态地址转向服务名路由。

## 新增能力

服务注册与配置中心。

## 技术范围

| 类别 | 技术 |
|---|---|
| 注册中心 | Nacos |
| 配置中心 | Nacos Config |
| 服务治理 | Spring Cloud |

## 架构变化

```text
Gateway
  -> Nacos service discovery
    -> Auth/User backend services
```

本阶段优先实现 Nacos 单节点可运行，再保留集群扩展 compose 计划。集群化可以在后续部署阶段继续强化。

## 开发步骤

1. 新增 Nacos Docker Compose 文件。
2. 后端服务接入服务注册。
3. Gateway 接入服务发现路由。
4. 把部分配置迁移到 Nacos Config，例如 JWT 配置、业务开关或登录限制。
5. 编写本地启动顺序说明。
6. 验证服务上下线对网关路由的影响。
7. 准备 Nacos 集群化草案，但不强行复杂化本阶段。
8. 更新 README 和 `docs/PROGRESS.md`。

## 验收标准

1. Nacos 可以启动并访问控制台。
2. Gateway 和后端服务可以注册到 Nacos。
3. Gateway 可以通过服务名转发请求。
4. 修改 Nacos 配置后，服务可以读取到配置。
5. 登录、用户管理、React、Vue 功能未回归。

## 完成定义

1. Nacos 服务注册可用。
2. Nacos 配置读取可用。
3. README 记录 Nacos 启动和访问方式。
4. 新增和修改代码已补充详细中文注释。
5. 如果需要保存版本，由用户手动提交并手动打 tag：`v0.6-nacos`；Codex 不自动执行提交、tag 或 push。
