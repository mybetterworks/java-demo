# v1.8 Docker And Kubernetes

## 目标

完成应用容器化，并提供 Docker Compose 和 Kubernetes 两套部署方式。

## 新增能力

容器化与 K8s 部署。

## 技术范围

| 类别 | 技术 |
|---|---|
| 容器 | Docker |
| 本地编排 | Docker Compose |
| 集群编排 | Kubernetes |
| 镜像 | 后端、Gateway、前端 |

## 功能范围

| 功能 | 说明 |
|---|---|
| 后端镜像 | 构建后端服务镜像 |
| Gateway 镜像 | 构建网关镜像 |
| 前端镜像 | Nginx 托管前端静态资源 |
| Compose 部署 | 本地一键启动主要服务 |
| K8s 部署 | Deployment、Service、ConfigMap、Secret |

## 开发步骤

1. 为后端服务编写 Dockerfile。
2. 为 Gateway 编写 Dockerfile。
3. 为 React/Vue 前端编写 Nginx 镜像配置。
4. 整理 full Docker Compose。
5. 编写 K8s namespace。
6. 编写 ConfigMap 和 Secret 模板。
7. 编写核心服务 Deployment 和 Service。
8. 编写 Ingress 或网关入口说明。
9. 验证 Compose 启动。
10. 验证 K8s 部署。
11. 更新 README 和 `docs/PROGRESS.md`。

## 验收标准

1. 应用镜像可以构建。
2. Docker Compose 可以启动主要应用和中间件。
3. K8s 可以部署 Gateway 和后端服务。
4. K8s 内部服务发现正常。
5. 外部可以通过入口访问前端和 API。
6. 配置和密钥不硬编码在镜像中。

## 完成定义

1. Compose 部署可验证。
2. K8s 部署可验证。
3. 部署文档已记录。
4. 新增和修改代码已补充详细中文注释。
5. 如果使用 Git，提交并打 tag：`v1.8-docker-k8s`。
