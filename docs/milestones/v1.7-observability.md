# v1.7 Observability

## 目标

引入 Prometheus、Grafana、SkyWalking 和 ELK，让系统具备指标、链路和日志三类可观测能力。

## 新增能力

可观测性。

## 技术范围

| 类别 | 技术 |
|---|---|
| 指标 | Prometheus |
| 看板 | Grafana |
| 链路追踪 | SkyWalking |
| 日志 | ELK |

## 功能范围

| 能力 | 说明 |
|---|---|
| 指标采集 | JVM、接口耗时、请求量、错误率 |
| 看板展示 | Grafana 展示关键指标 |
| 链路追踪 | 查看 Gateway 到后端的调用链 |
| 日志检索 | 应用日志进入 Elasticsearch 并可查询 |

## 开发步骤

1. 后端暴露 Actuator 指标。
2. 新增 Prometheus Docker Compose。
3. 新增 Grafana Docker Compose 和基础 dashboard。
4. 接入 SkyWalking Agent。
5. 配置服务名和链路采样。
6. 新增 ELK 或 Filebeat/Logstash 采集链路。
7. 统一日志格式，包含 trace id。
8. 验证一次登录请求的指标、链路和日志。
9. 更新 README 和 `docs/PROGRESS.md`。

## 验收标准

1. Prometheus 可以采集应用指标。
2. Grafana 可以展示 dashboard。
3. SkyWalking 可以看到一次登录请求链路。
4. ELK 可以检索应用日志。
5. 日志中可以关联 trace id。
6. Gateway、后端服务都有可观测数据。

## 完成定义

1. 指标、链路、日志三条路径均可验证。
2. 访问地址、账号、端口已记录。
3. 新增和修改代码已补充详细中文注释。
4. 如果使用 Git，提交并打 tag：`v1.7-observability`。
